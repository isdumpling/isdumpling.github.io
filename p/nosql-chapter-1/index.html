Get familiar with NoSQL Database

Understanding NoSQL Database

the important features of NoSQL

  • Schema Flexibility: allows for the storage of various data types
  • Horizontal Scalability: scaled out by distributing data across multiple servers
  • High Availability and Fault Tolerance: Build-in replication and sharding features
  • Performance: Optimized for specific use cases
  • Diverse Data Models: Support various data models
  • Cost-Effectiveness: The ability to scale out using commodity hardware and no need to invest the expensive, high-end servers

Introduction to NoSQL

  • Two types of databases
    • relational databases
    • non-relational databases, often called NoSQL databases
  • A NoSQL database is used to store large quantities of complex and diverse data
  • MongoDB is one of the most established NoSQL databases
    • data aggregation
    • ACID
      • Atomicity
      • Consistency
      • Isolation
      • Durability
    • horizontal scaling
    • charts

Relational Database VS NoSQL

  • Relational Database Management Systems
    • store structured data
    • in the form of tables that consist of rows and columns
    • have relationships with other tables
  • NoSQL Database Management Systems
    • store unstructured and semi-structured data
    • store the data without a schema and support dynamic schema
FeatureRelational DatabaseNoSQL Database
Schema1. follows a rigid schema 2. have a definition of all the desired columns and their types1. does not impose a rigid schema 2. store the unstructured data with dynamic structures
Data model/Storage Structure1. data is stored in tables 2. each record is stored as a row1. data is stored in different formats 2. storage structure are documents, graphs, key-values and wide columns
Normalizationused to remove duplicate data and avoid data anomaliesfocus more on fast data retrieval and the data can be normalized
Scalinghard to scale and generally scaled verticallyboth vertical and horizontal scaling
